Project Overview

PyShaft is a high-level web automation framework built on top of Selenium. It emphasizes Lazy Execution, Fluent API design, and Reliability through an automated wait engine.

Lazy Execution

Commands are queued and only resolved when a terminal action or assertion is triggered.

Fluent API

Chainable methods like w.click(role, button).nth(1) for readable tests.

Smart Waits

Automatic handling of overlays, animations, and staleness via the internal WaitEngine.

Architecture

The core logic is split between the Action Pipeline and the Wait Engine.

graph TD A[User Script] --> B[WebEngine/w instance] B --> C[Locator Object Created] C --> D{Terminal Action?} D -- Yes --> E[action_runner.py] D -- No (Chain) --> C E --> F[WaitEngine] F --> G[Selenium Driver] G --> H[DOM Interaction]

The Lazy Pipeline

PyShaft doesn't click or type immediately. It builds a "Pending Locator" state.

Execution Flow:

  1. Registration: WebEngine._register_action(loc) saves the locator to _pending_locator.
  2. Flushing: Any subsequent action or assertion calls engine.flush().
  3. Resolution: Locator.execute() builds the final selector and invokes the actual interaction.

Project Structure

Path Purpose
pyshaft/core/ The "Brain". Action runners, WaitEngine, and Driver Factory.
pyshaft/web/ The "Body". Interaction modules (inputs, navigation, js_helpers).
pyshaft/web/__init__.py The "Face". Entry point for the unified fluent API.
tests/ Integration and unit tests. Always run with -p no:seleniumbase.

Adding New Actions

To add a new feature (e.g., from SeleniumBase), follow these 4 steps:

1. Implement Logic in web/

Create the function in the relevant module (e.g., inputs.py).

def my_new_action(locator: str, value: str) -> None:
    def _logic(element: WebElement) -> None:
        element.do_something(value)
    run_action("my_action", locator, _logic)

2. Update locators.py

Add a terminal method to the Locator class and update the execute() dispatcher.

# Inside Locator class
def my_action(self, value):
    self._action = "my_action"
    self._action_args = {"value": value}
    return self

# Inside execute()
elif self._action == "my_action":
    from pyshaft.web.module import my_new_action
    my_new_action(selector, self._action_args.get("value"))

3. Expose in web/__init__.py

Add a chainable method to WebEngine and an alias in __all__.

def my_action(self, value, locator_type=None, val="", **filters):
    loc, _ = self._build_loc(locator_type, val, **filters)
    loc.my_action(value)
    return self._register_action(loc)

MFA & Security

PyShaft supports TOTP-based 2FA bypass using pyotp.

Implementation Example:

w.enter_mfa_code("JBSWY3DPEHPK3PXP", id_, "otp-field").execute()

Note: Ensure the TOTP key is stored securely in environment variables or pyshaft.toml.

Master Dictionary (MD) Updates

For multi-language support (SeleniumBase pattern), follow this mapping workflow:

1. Modify master_dict.py

Add the new method name and its translations to the MD.md dictionary using the language index keys (0=EN, 1=ZH, 9=ES).

2. Update Language Files

Update chinese.py, spanish.py, etc., to wrap and return the new core method.

Configuration

PyShaft configuration is hierarchical:

  1. config.py: Contains global default constants.
  2. pyshaft.toml: Root config file for local overrides (e.g., downloads_dir).
  3. Environment Variables: System-level overrides for CI/CD pipelines (prefixed with PYSHAFT_).

Troubleshooting

Issue: SessionNotActiveError

Ensure you are calling session_context.start(driver) or running via the pytest plugin with a supported browser.

Plugin Collision: SeleniumBase

If you have SeleniumBase installed, use pytest -p no:seleniumbase to prevent double-hook registration.

Issue: Unclickable Overlays

The WaitEngine handles most cases. If an element is blocked, try w.click_all(...) or use the JS-direct js.js_click(...).
